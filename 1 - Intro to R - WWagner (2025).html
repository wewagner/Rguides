In this document, you will get to know R as a programming language and RStudio as a coding environment, you will explore some basic operations, and understand how to import and explore different kind of datasets within RStudio. Once we are familiar with these operations, we move on to basic data exploration, to data manipulation like recoding, merging, and aggregating, and finally, we advance to assessing distributions and relationships visually.

2. Introduction to R

R is a language as well as an environment for statistical computing and graphics. We will use it for data manipulation, calculation, analysis and visualization. You will quickly learn how to “communicate” in R language, how to assign values, create objects, and understand syntax. While other software can achieve similar tasks, R is widely used in both soft and hard sciences and has two added benefits. First, R is completely open source and free. Second, it is a crowd sourced language. Users are constantly updating and revising the language. And users are constantly creating new libraries (tools that make certain data tasks easier to achieve). R therefore has some of the most up-to-date and innovative functionality and methods available to its users, should they know where to look.

4. Getting Familiar with RStudio

Before anything else, you must become familiar with RStudio. You might have worked with data analysis software like Excel, SPSS, JMP, or Stata prior to this point. R will introduce you to a new way of working with data. In comparison to Stata, RStudio will allow you to hold multiple data sets in memory and work with them at the same time (whereas Stata only allowed us to open one data set at a time and made operations such as merging more cumbersome).

Also different from SPSS, JMP, or Stata, R does not offer a mode of “clicking through” to conduct data analysis. While Stata allowed us to generate visualizations or analyze data via integrated pop-up windows - in RStudio we will only use direct code. We will write and save our code in RScript files that will remind you of Stata .do files. Yet, R language will become more intuitive to you over time.

The image below shows the 4 primary components of the RStudio interface.

In the upper-left corner is the source pane. This is the primarily location where most of your work will take place. You will write and edit collections of code - or R Scripts - here. When working in the source pane, your code will not compile until you tell it to run; this allows you the flexibility to work at your own pace, as well as to save your work.

In the lower-left corner is the console, or the command window. The console is the powerhouse of the software; this is where R actually evaluates code. While you can type code directly into the console and receive immediate results, it is advisable to stick to the source pane while you are learning how to use R. Running code in the source pane will automatically produce output in the console pane.

The upper-right quadrant contains the Environment and History tabs. The Environment tab displays a list of all the data objects that you have defined in your current R session, as well as some basic details about the data (such as the number of observations and variables in each). The History tab contains an archive of all the commands you’ve run in the current session.

Finally, the lower-right quadrant holds a number of helpful navigation tabs. The “Files” tab displays your hard drive’s own file directory for easy access. The “Plots” tab will show the plots and visualizations you have created in your current R session. The “Packages” tabs shows a list of all the packages currently installed, as well as an indication of whether or not that are loaded in the current session. The “Help” tab is, unsurprisingly, the help menu.

A “cheat sheet” for the RStudio IDE can be found here https://posit.co/wp-content/uploads/2022/10/rstudio-ide-1.pdf. As an open source software, there are many helpful resources available, especially for libraries constructed using R (more on this shortly), for reference https://posit.co/resources/cheatsheets/?type=posit-cheatsheets&_page=1/.

4.1 R Scripts

You can work in R Studio in two ways. First, you can write calculations and commands directly into the “Console” area. Try right now, writting the following in your console:

x <- c(1:100)

You should observe that you have now created a vector named x that contains a list of values from 1 through 100.

Second, you can write and save your commands in a document called script. Click on the small document sign with a plus on the top left corner of the window. And write the command in the script instead. You now have written code that can be easily saved, reopened and replicated at a later point. Scripts can (and often should) include useful information besides the code itself. For example, you can provide commentary using # symbols:

# My first code snippet in R:
x <- c(1:100) # Creating a vector from 1:100

4.2 R Packages

Within R, functions and tools are often implemented as packages. Imagine packages as a suite of tools with functions that simplify your life as a researcher. Instead of writing yourself 5 plus 5, you can use the function sum():

# Type the following two ways into your console and your console will return value 10
5 + 5
## [1] 10
# Or:
sum(5, 5)
## [1] 10

Packages allow advanced users of R to contribute statistical methods and computing tools to the general users of R. These packages are reviewed and vetted before being made available for public use; they are often frequently updated as well. You will soon become familiar with the most common packages and most widely used functionalities of popular libraries. We refer to packages when installing new tools using the command install.package. Once installed, we can call on the installed tools by calling libraries at the beginning of every R session.

Use the code “install.packages()” with the package name in ““. Once you have set up your RStudio, install the following packages:

  • psych
  • Rcpp
  • purrr
  • tidyverse
  • ggplot2
  • stargazer

These are all common “libraries” we will be using throughout the course. Call the packages later on with the command “library()”. Note that each package has to be installed on your computer only ONCE. In contrast, you need to call the library in every new session you intend to work with it.

4.3 R Markdown

The document you are reading is a R Markdown document. Markdown is a simple formatting syntax for authoring HTML, PDF, and MS Word documents. Markdown is a tool for converting plain text into formatted text that utilizes the markdown syntax in order to combine formatted text with code in a single document. It allows you to simultaneously 1. save and execute code, and 2. produce high quality documents that include both code and text.

For the purposes of our tutorial, R Markdown allows us to include code chunks and the text that helps explain them in an easy-to-read manner. For your own use, R Markdown will allow you to write documents and reports that include traditional formatted text, as well as the data visualizations you make in class, and present them both together in a high quality professional document. For example, we can have Markdown documents present text, code snippets, and the resulting output from data analysis; or we can omit the output while displaying the code; and vice versa have it display the output without showing the code needed to generate it. It is truly genius if you want to avoid formatting latex documents directly.

For more details on using R Markdown see http://rmarkdown.rstudio.com. R Markdown has to be installed as well, but you can do so within Rstudio. Please ask your TA or instructor for further guidelines on this if you would like to use R Markdown.

When you click the Knit button a document will be generated that includes both content as well as the output of any embedded R code chunks within the document. You can embed an R code chunk like this:

summary(cars) # From the cars package
##      speed           dist       
##  Min.   : 4.0   Min.   :  2.00  
##  1st Qu.:12.0   1st Qu.: 26.00

You can also embed plots, for example:

If using R Markdown, there are more things to learn. We can add the echo = TRUE parameter to a code chunk to print the R code that generated the plot (or we can add echo = FALSE to prevent printing the code chunk).

5.2 Objects

R is an “object oriented” programming language. Put simply, R uses objects to store attributes. Objects are created by assigning an attribute to them via the “<-” operation. You can always view the attribute of an object by typing the object name. We first assign the object with the name object some value and then print the value it contains. R does not automatically print the value when you assign it. You can force this by typing the object name or print.

object <- 25 + 2
object
## [1] 27
# or
print(object)
## [1] 27

Now that the object is in memory, we can perform opertations with it:

object * 2
## [1] 54
object * object
## [1] 729
# Save the result of an operation in a new object 
z <- object * object
z
## [1] 729

We can overwrite the object content:

z <- 17
z
## [1] 17

We can remove created objects simply by typing rm():

# Removing single objects:
rm(object)
rm(z)
# Or removing all objects:
rm(list=ls()) # Command do delete everything in your working environment (be careful!)

When choosing names for your objects, be consistent and intuitive. Usually we would refer in object names to their content. If you are storing results from an OLS model, we might call the object ols1 and ols2 and so on. Be aware of case sensitivity, lower case names are easier to work with. Additionally, avoid names that exist as functions in R, such as list or mean or sum. These are reserved for existing functions, and R can get confused if it finds objects with identical names.

5.3 Vectors (Lists)

objects and z, which we created previously are numeric vectors, that each hold one single value But we can store into our objects more than one value using a command called c() (for concatenate). For example:

# w is a vector with values from 1 through 5
w <- c(1,2,3,4,5)
print(w)
## [1] 1 2 3 4 5
w <- c(1:5) # a better way for including the values from 1 through 5
z <- c(1:100) # values 1 through 100
v <- c(100:1) # values 100 through 1 (descending order)
t <- seq(1,100,7) # values 1 through 100, in increments of 7

Vectors are essentially like a single column of data. We will see later on why these may be useful. Vectors can either hold only numbers, or only text (but not both together):

myname <- "Wendy" # text is stored between quotation marks
yourtas <- c("Wendy","Salma","Marcia","Jesus")
years <- c(2019, 2020, NA, 2020)

Note the use of NA. This represents missing data. We will learn soon how to work with missing data soon. Now note which objects we currently hold in memory:

ls()
## [1] "myname"  "t"       "v"       "w"       "years"   "yourtas" "z"

5.4 Dataframes

data.frames are perhaps the most common type of object when starting to work in R. They are essentially tables of data with two dimensions: rows (dimension 1) representing observations and columns (dimension 2) representing variables. They most resemble Excel sheets, spss datasets, stata datasets, and so on. We can create a data.frame from two vectors (if they are the same length and in the same order):

mydata <- data.frame(yourtas, years)

Use the command “View()” to open and view the dataframe as a data table.

View(mydata)

A dataframe is essentially a matrix consisting of rows and vectors. You can generate a matrix with any number of columns (variables) and rows (observations). Observations can be countries, individuals, survey respondents, country-years and so on. And your data can be organized in ‘long’ or ‘wide’ format (but more on that later). Note how the following code snippet creates a new object called ‘newdata’ which is assigned a dataframe with 2 rows and 3 columns. In this case, the dataframe is empty as we have not provided any information to fill it.

newdata <- data.frame(matrix(NA, nrow = 2, ncol = 3))

Now you are ready to import and explore some actual data in R. Before moving on to the next part of the tutorial, make sure to clean your working environment:

rm(list=ls())

7. Setup and Importing Data

7.1 Setup a Working Directory

The working directory is the location where files are accessed and saved within a R session. Normally, the working directory is set at the beginning of every R file. The working directory should be set and the data loaded at the beginning of each tutorial or work session. If no directory is provided, R will likely default to saving output in your Downloads or Documents folder. Make sure to choose the directory, i.e. the folder location in your computer, where R can find the data it should load and where it should save any output we generate such as tables or figures.

The setwd() function can be used with the directory path. For example, setwd(“C:/Directory_to_folder/”).

Note: Forward slashes are used in place of backward slashes for directory paths.

getwd() # Returns the path to the folder currently set as working directory

Importing Foreign Data

R prefers data to be in CSV format. This is not always the case. You can also load and import data from Excel, SPSS, or Stata files. Here are some examples on how to do so:

For spss files, we need to explicitly ask R to import the dataset as a data.frame ( to.data.frame = T ), we can choose if we want to use existing value lables ( use.value.labels = T/F ) and if we want R to convert prespecified missing values to NA ( use.missings = T ).

# install.packages("foreign") # If this is the first time ever using this package
library(foreign) 
breakfast.spss <- read.spss("breakfast.sav",
to.data.frame = T, use.value.labels = T, use.missings = T)

This can also be achieved using the haven package:

# install.packages("haven")
library(haven)
breakfast.spss <- read_sav("breakfast.sav")

For Stata files: read.dta can directly import State data files

library(foreign) # we don't actually have to do this again, we call it only once at the beginning of the session
statacars <- read.dta("auto.dta") # import sample file auto.dta from stata

For Excel files:

# install.packages("readxl") # turns to CRAN (R online archive) to install package
library(readxl)
sample.Excel <- read_xlsx("randomdata.xlsx")

If an Excel file has multiple sheets, or we want a specific range, we can specify that as well. At times data in Excel files will only begin in a certain column or certain row:

sample.Excel10 <- read_xlsx("randomdata.xlsx", sheet = "Sheet1",
range = "A1:C11") # first ten rows only

7.3 Exporting Data

At the end of a session you might have made many manipulations to your data. You might have assigned values to missing values, you might have created new variables, or renamed old ones. Never save this data in the name of the original file. Instead, export your final dataset as a new file for later reference.

To export as a CSV, write.csv() is a useful baseR command. It’s downside is it exports NA as NA, rather than empty cells, and it exports row names as well. However, both can be over written:

write.csv(cars, "cars.csv", na = "", row.names = F)

Another good option is write_csv() from the readr package, which is often faster and does not export row names:

library(readr)
write_csv(cars, "cars.csv", na = "")

CSV should be the preferred way of exporting your data. Yet, you can also export files as Excel (or other data types):

library(writexl)
write_xlsx(cars, "cars.xlsx")
